# Trabajo Práctico Final - Programación II (2.603) - 2025

## Tema
El trabajo práctico final consistirá en el desarrollo de una **aplicación completa**, utilizando **Java** y una **base de datos relacional**, sobre el **mismo tema del primer trabajo práctico del año**.  
En caso de que el grupo desee trabajar sobre un tema distinto, deberá **consultarlo previamente** y presentar una **propuesta clara y fundamentada** del nuevo tema.

## Requisitos Técnicos

### Actores (Usuarios del Sistema)
El sistema deberá contar con **al menos 3 actores distintos**, es decir, tres tipos diferentes de usuarios con funcionalidades diferenciadas (por ejemplo: Administrador, Usuario, Invitado).

### Frameworks
Se deberán utilizar **al menos 2 frameworks de Java**, tales como:
- Spring Boot / Spring MVC / Spring Data JPA
- Hibernate
- Thymeleaf  
  *(u otros frameworks equivalentes, previa aprobación del docente)*

### Base de Datos
- Deberá ser **relacional** (por ejemplo, MySQL, PostgreSQL o H2).
- Se deberá diseñar el **modelo entidad-relación** correspondiente.
- La aplicación deberá incluir la **persistencia completa de datos** (ABM funcionales).

### Arquitectura del Proyecto
- El proyecto deberá estar organizado en **capas** (por ejemplo: Controlador, Servicio, Repositorio, Modelo).
- Se deben cumplir buenas prácticas de programación y separación de responsabilidades.
- Se debe respetar los pilares de la POO y los principios SOLID.

### Control de Versiones
- El proyecto deberá estar alojado en **GitHub**.
- Todos los integrantes del grupo deberán figurar como **colaboradores activos** del repositorio.

## Evaluación Documentada

La entrega deberá incluir un **documento de evaluación** (en formato PDF) con la siguiente estructura:

### Portada
- Nombre de la materia
- Docente
- Integrantes del grupo
- Curso y año
- Título del proyecto
- Link al repositorio de GitHub (Debe contener el proyecto completo y ejecutable)

### Descripción general del proyecto
- Tema elegido
- Objetivos del sistema
- Descripción de los actores y sus roles

### Arquitectura y desarrollo
- Descripción de las capas del sistema
- Explicación de la estructura del proyecto
- Descripción de las tecnologías y frameworks utilizados
- Explicación de las clases principales y sus responsabilidades
- Capturas o ejemplos de código relevantes

### Casos de uso y diagramas
- Diagrama de casos de uso con los actores involucrados
- Breve descripción de cada caso de uso

### Base de datos
- Diagrama entidad-relación
- Descripción de las tablas y relaciones
- Ejemplo de datos cargados (opcional)

### Manual de usuario
Se deberá incluir un **manual de uso para cada actor del sistema**, explicando:
- Cómo acceder al sistema
- Qué funcionalidades tiene cada usuario
- Ejemplos o capturas de pantalla de las pantallas principales

## Criterios de Evaluación
- Cumplimiento de los requisitos técnicos
- Correcta aplicación de los frameworks seleccionados
- Organización y estructura del código
- Claridad en la documentación
- Diseño adecuado de la base de datos
- Funcionamiento completo de las funcionalidades requeridas
- Interfaz clara y funcional
- Trabajo colaborativo y uso efectivo de GitHub
- Defensa oral personal (individual) sobre cualquier parte o aspecto del proyecto
